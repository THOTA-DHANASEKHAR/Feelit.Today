alias

SQL ALIASE / AS


In SQL, an alias is a temporary name or label assigned to a table or column in a query's result set. Aliases are used to make column or table names more readable, to provide alternative names for columns, and to simplify complex queries.
Aliases are created using the `AS` keyword in the `SELECT` statement.


Here are two common use cases for aliases in SQL:

  1. Column Alias
  2. Table Alias

Table-1 Name: employees:
id name salary dept_id
1 Alice 50000 1
2 Bob 60000 2
3 Charlie 55000 1
4 David 70000 2

Table-2 Name: departments:
dept_id dept_name
1 HR
2 Finance


1. Column Alias:

Column aliases are used to provide alternative names for columns in the result set. They are helpful when you want to change the name of a column in the output to make it more understandable or when you need to provide a name for the result of an expression or calculation.

Syntax: The basic syntax of the Column Alias as follows:

Syntax: Copy
SELECT column_name AS alias_name 
FROM table_name;

Example for Column Alias:
Example: Copy
SELECT 
id AS "emp id", 
name AS "my full name" 
salary AS "my salary"
FROM employees;

Result:
emp id my full name my salary
1 Alice 50000
2 Bob 60000
3 Charlie 55000
4 David 70000

In this example, the result set will have column headers "emp id", "my full name" and "my salary" instead of "id", "name" and "salary"



2. Table Alias:

Table aliases are used to provide a shorter or more convenient name for a table in the query. They are commonly used when you have to join multiple tables, and the table names are long or complex.

Syntax: The basic syntax of the Table Alias as follows:

Syntax: Copy
SELECT alias_name.column_name 
FROM table_name AS alias_name;

Example for Table Alias:
Example: Copy
SELECT e.id, 
e.name, 
d.dept_name
FROM employees AS e 
JOIN departments AS d ON e.department_id = d.department_id;

Result:
id name dept_name
1 Alice HR
2 Bob Finance
3 Charlie HR
4 David Finance

In this example, "e" and "d" are table aliases for the "employees" and "departments" tables, respectively, making the query more concise and readable.



To summarize: Aliases can also be used with aggregate functions, subqueries, and more complex expressions. They improve query readability and maintainability and are especially valuable in large and complex SQL queries. Additionally, aliases are often required when performing self-joins, where a table is joined with itself.
