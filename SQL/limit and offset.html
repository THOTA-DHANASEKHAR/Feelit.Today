SQL LIMIT AND OFFSET CLAUSE


In MySQL, the LIMIT and OFFSET clauses are used to control the number of rows returned by a query and to skip a specified number of rows. These clauses are often used for pagination and to retrieve a subset of rows from a result set.

  • LIMIT : clause specifies the maximum number of rows to return.
  • OFFSET : clause determines how many rows to skip before starting to return rows.

Syntax: The basic syntax of the LIMIT and OFFSET statements is as follows:

Example: Copy
SELECT column1, column2, ... 
FROM table_name 
LIMIT number_of_rows 
OFFSET offset_value;

1. LIMIT: To retrieve the top 2 highest salary employees details, you can use Limit clauses.

Example: Copy
SELECT id, name, salary 
FROM employees
ORDER BY salary
LIMIT 2;

Result:
id name salary
4 David 70000
2 Bob 60000

LIMIT 1 : LIMIT 1 will display 1 record from top.



2. OFFSET: To retrieve the 3nd highest salary employee details, you can use OFFSET & Limit clauses.

Example: Copy
SELECT id, name, salary 
FROM employees
ORDER BY salary
LIMIT 1
OFFSET 2;

Result:
id name salary
3 Charlie 55000

LIMIT 1 OFFSET 2 : OFFSET 2 will skip top 2 rows from top and LIMIT 1 will display 1 record from top.



To summarize: the LIMIT clause restricts the number of rows in each result set, and the OFFSET clause determines where to start retrieving rows from the result set. This can be used for implementing pagination, where you can adjust the LIMIT and OFFSET values to navigate through a large result set in a user-friendly way.
