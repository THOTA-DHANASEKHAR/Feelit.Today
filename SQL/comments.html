SQL comments


SQL supports both single-line and multi-line comments to add notes or explanations within SQL code. The syntax for comments varies between database systems. Common ways to add comments in SQL include:


There are two types of SQL comments:

Single-line comments: Single-line comments start with two hyphens (--) and end at the end of the line. Any text between the -- and the end of the line is ignored by the SQL parser.

Multi-line comments: Multi-line comments start with a slash and an asterisk (/*) and end with an asterisk and a slash (*/). Any text between the /* and the */ is ignored by the SQL parser.


Here are some examples of SQL comments:

Example: Copy
-- This is a single-line comment.
                
/*
This is a multi-line comment.
It can span multiple lines.
*/

SELECT * FROM customers; -- This query selects all rows from the `customers` table.

UPDATE customers 
SET name = 'John Doe' 
WHERE id = 1; /* This query updates the name of the customer with the ID of 1 to 'John Doe'. */;

Comments can be used to:

  • Explain the purpose of a SQL statement or section of code.
  • Disable SQL statements without deleting them, which can be useful for debugging or testing purposes.
  • Add additional information to SQL code, such as the date the code was written or the author of the code.

To summarize: Comments are a good way to make your SQL code more readable and maintainable. It is especially important to use comments when writing complex SQL queries or when working on code that will be shared with other developers.
