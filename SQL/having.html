SQL HAVING CLAUSE


The HAVING clause in SQL is used in combination with the GROUP BY clause to filter the result set based on aggregate functions or computed values.

It is typically used when you want to apply conditions to groups of rows rather than individual rows.

The HAVING clause is mainly used in the context of aggregate queries, such as when you want to perform calculations on groups of data and filter the results based on those calculations.


Syntax: The basic syntax of the HAVING statement is as follows:

Syntax: Copy
SELECT column1, column2, ... 
FROM table_name 
GROUP BY column1, column2, ... 
HAVING condition;

1. Group calculation: You can use GROUP BY to perform Aggregations for each group of values.

If you want to find the total salary spent on each department, and only retrieve the results for dep_id's with a total_salary greater than 120000, you can use the GROUP BY & HAVING clauses:

Example: Copy
SELECT dept_id, 
sum(salary) as total_salary
FROM employees
GROUP BY dept_id
HAVING SUM(salary)>120000;

Result:
dept_id total_salary
2 130000

To summarize: The HAVING clause is particularly useful for performing calculations on aggregated data and filtering the results based on those calculations. It allows you to focus on subsets of data that meet specific criteria after grouping the data using the `GROUP BY` clause.
